# Users

Тестовое задание - Приложение для управления и просмотра базы пользователей.

## Функционал

- Просмотр списка пользователей: Отображение базы с использованием виртуализации.
- Просмотр отделов: Статистика сотрудников по группам и визуализация структуры компании.
- Сортировка данных: Сортировка по имени, фамилии, email и группам.
- Управление данными: Возможность удаления и добавления записей прямо из таблицы.
- Адаптивная верстка: Поддержка мобильных устройств.

## Используемые технологии

- React 19
- TypeScript
- Vite
- Sass (SCSS Modules)
- React Router
- React Virtuoso
- Lucide React
- JSON Server
- ESLint / Stylelint / Prettier

## Установка и запуск проекта

1.  Клонировать репозиторий: `git clone https://github.com/sdlmdev/Users`
2.  Установить зависимости: `npm install`
3.  Сгенерировать базу данных (при необходимости): `npm run db:gen`
4.  Запустить проект: `npm run dev` (запускает Vite и API сервер одновременно)

## Выводы по проектированию

Процесс разработки данного проекта сочетал в себе классическое ручное проектирование и использование возможностей LLM.

### Проектирование при помощи LLM

- **Скорость прототипирования**: LLM позволила в кратчайшие сроки сгенерировать сложную структуру компонентов и базовые стили.
- **Рефакторинг и декомпозиция**: Использование LLM для декомпозиции позволило провести рефакторинг, сохранив чистоту кода и логики.

### Итог

Использование LLM значительно ускоряет рутинные операции по созданию UI-компонентов и базовой стилизации. Однако финальная сборка, оптимизация производительности (виртуализация) и проектирование межкомпонентного взаимодействия требуют осознанного ручного контроля для обеспечения стабильности и масштабируемости приложения.
